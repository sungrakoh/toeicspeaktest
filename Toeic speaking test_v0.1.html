📱 모바일에서 PC와 똑같이 쓰기

아래 순서대로 1분이면 휴대폰(크롬·사파리)에서 여는 HTTPS 링크를 만들 수 있습니다.

왜 파일을 그냥 못 여나요?
① 휴대폰 브라우저는 PC의 로컬 파일(file://)을 열 수 없고, ② 마이크 녹음 기능은 보안 주소(https://…)에서만 작동합니다. 그래서 index.html을 무료 웹 호스팅에 올려 https 링크로 열어야 PC와 100% 동일하게 동작합니다.

준비물

앞서 받은 index.html 파일 1개 (이미 모바일 대응 완료). 이 파일만 올리면 됩니다.

방법 A — 가장 빠름: Netlify Drop 추천 · 끌어다 놓기

  1. PC 브라우저에서 app.netlify.com/drop 접속
  2. 화면의 점선 영역에 index.html 파일을 끌어다 놓기 (drag & drop)
  3. 잠시 뒤 https://○○○.netlify.app 형태의 주소(링크)가 생성됩니다
  4. 그 링크를 복사 → 아래 QR 만들기에 붙여넣어 휴대폰으로 스캔하거나, 카톡/문자로 폰에 보내 열기

※ 링크를 영구 보관하려면 무료 가입(이메일) 후 그대로 두면 됩니다. 가입 전에는 임시로 유지됩니다.

방법 B — 영구 무료: GitHub Pages

  1. github.com 무료 가입/로그인 → New repository로 새 저장소 생성(예: toeic), Public 선택
  2. Add file → Upload files로 index.html 업로드 후 Commit
  3. Settings → Pages → Branch를 main / /(root)로 지정 후 Save
  4. 1~2분 뒤 https://(아이디).github.io/toeic/ 주소가 나옵니다 → 휴대폰에서 열기

방법 C — 대안: Cloudflare Pages / tiiny.host

Cloudflare Pages(pages.cloudflare.com) 또는 tiiny.host(tiiny.host)에 동일하게 index.html 업로드 → https 링크 생성. 단일 파일 업로드에 특화되어 간단합니다.

🔳 링크 → QR 코드 만들기

위에서 만든 https 링크를 붙여넣고 버튼을 누르면 QR이 생성됩니다. 휴대폰 카메라로 스캔하세요.

✅ 휴대폰에서 처음 열 때

  • 마이크 허용: ‘시작’을 누르면 한 번 물어봅니다. 허용을 누르면 이후 다시 묻지 않습니다.
  • iPhone은 Safari 사용을 권장합니다. (실시간 인식은 iOS 미지원 → 기본값 ‘온디바이스 AI’ 또는 ‘직접 입력’ 사용)
  • 온디바이스 AI 인식은 최초 1회 모델(약 40MB)을 내려받으므로 Wi-Fi에서 첫 채점을 권장합니다. 느린 기기는 ‘자동 인식 끔’ 후 결과 화면에서 답변을 직접 입력해 채점해도 됩니다.
  • 홈 화면에 추가하면 앱처럼 쓸 수 있습니다: Safari 공유 → 홈 화면에 추가 / 크롬 ⋮ → 홈 화면에 추가.
  • 프라이버시: 녹음·답변·점수는 휴대폰 브라우저 안에서만 처리되며 외부로 전송되지 않습니다.
